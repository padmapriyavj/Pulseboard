# Pulseboard

> **PulseBoard** is a real-time IoT analytics dashboard that collects, visualizes, and analyzes telemetry data from IoT devices. It features a modern, scalable architecture leveraging event streaming, caching, timeseries storage, and a responsive frontend.

---

## 🎯 Key Features

- 📈 Real-time telemetry visualization (charts, dashboards, metrics)
- 🔔 Live alerts for abnormal readings
- 📨 Kafka-based telemetry event ingestion pipeline
- ⚡ Redis cache for hot metrics
- 🗄️ PostgreSQL/TimescaleDB for historical data analysis
- 🔒 JWT-based authentication and role-based access control
- 📦 Dockerized architecture (Docker Compose support)
- 🚀 CI/CD workflow (GitHub Actions)
- ☁️ Cloud-ready (AWS / Render / Railway)


---

## 🛠️ Tech Stack

| Layer | Tech |
| ----- | ---- |
| Frontend | React + TypeScript + Tailwind CSS |
| Backend API | Node.js + Express or NestJS + GraphQL (or REST) |
| Real-time Updates | WebSocket / Server-Sent Events |
| Messaging | Apache Kafka |
| Caching | Redis |
| Database | PostgreSQL / TimescaleDB |
| Auth | JWT |
| DevOps | Docker, Docker Compose, GitHub Actions |
| Deployment | AWS Free Tier or alternatives (Render, Railway) |
